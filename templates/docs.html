# 1. ¿Qué es Secure the Box?

Secure the Box es una plataforma de entrenamiento Blue Team orientada a analistas SOC. El objetivo es aprender técnicas de hardening y defensa aplicadas a máquinas vulnerables en un entorno controlado.

Cada máquina presenta una serie de configuraciones inseguras que debes identificar y corregir. Una vez aplicadas, un script de evaluación verifica tu trabajo y, si todo está correcto, registra tu logro automáticamente y te otorga los puntos correspondientes.

// FLUJO BÁSICO

  1. Regístrate y acepta la normativa.
  2. Descarga y despliega la máquina con Docker.
  3. Aplica las medidas de hardening.
  4. Ejecuta el script de evaluación — si todo está correcto, genera una FLAG.
  5. Canjea la FLAG desde tu panel antes de que caduque (30 min).
  6. Comprueba tus puntos en el ranking global.

# 2. Registro y configuración de cuenta

Editar perfil

Iniciar y cerrar sesión

Usa los enlaces del menú de navegación. La sesión se mantiene activa hasta que cierres sesión manualmente.

# 3. Sistema de FLAGS

La validación de máquinas se basa en FLAGS firmadas criptográficamente. Cuando el script de evaluación verifica que el hardening es correcto, genera una FLAG única con HMAC-SHA256 y la deposita automáticamente en la plataforma. Tú solo tienes que canjearla desde tu panel.

# Formato de FLAG
STB-<maquina>-<timestamp>-<hmac>

# Ejemplo
STB-linux-hardening-1716048000-a3f8b2c14d5e6789
→

Generación automática: el script crea y deposita la FLAG si el hardening pasa todos los checks. No tienes que copiarla manualmente del script.

→

Caducidad: cada FLAG es válida durante 30 minutos desde su generación. Pasado ese tiempo queda invalidada.

→

Un solo uso: una FLAG solo puede canjearse una vez. No es posible reutilizarla ni compartirla con otros usuarios.

→

Canjeo: accede a tu panel (/panel) y pega la FLAG en la sección correspondiente para obtener los puntos.

# 4. Normativa y Reglas de Uso

Antes de acceder al panel debes leer y aceptar la Normativa de Uso. Recoge las reglas de conducta de la plataforma y las restricciones sobre las actividades permitidas.

! ADVERTENCIA

  • Está terminantemente prohibido atacar la infraestructura de la plataforma.
  • Solo puedes interactuar con las máquinas asignadas en tu entorno local.
  • El intercambio o venta de FLAGS o API Keys está prohibido y conlleva la expulsión permanente.
  • Toda la actividad queda registrada y puede ser auditada.

Consulta la normativa completa en /normativa.

# 5. Despliegue de Máquinas

Cada máquina se distribuye como un entorno Docker autocontenido. Necesitas tener instalados Docker y Docker Compose en tu sistema.

Paso 1 — Descarga el repositorio de la máquina

git clone <url-del-repositorio-de-la-maquina>
cd <nombre-de-la-maquina>

Paso 2 — Levanta el entorno

docker-compose up -d

El flag -d arranca los contenedores en segundo plano.

Paso 3 — Comprueba que está activa

docker-compose ps

Todos los servicios deben aparecer con estado Up.

Paso 4 — Detener la máquina (cuando termines)

docker-compose down

// REQUISITOS DE SISTEMA

  • Docker Engine >= 24.x
  • Docker Compose >= 2.x
  • Python >= 3.10 (para el script de evaluación)
  • Conexión a Internet para descargar imágenes Docker

# 6. Script de Evaluación

El script check_hardening.py incluido en el repositorio de cada máquina realiza dos cosas: verifica que las medidas de hardening son correctas y, si todo pasa, genera y deposita automáticamente una FLAG firmada en la plataforma.

Uso básico

python3 check_hardening.py

No se necesita ningún parámetro adicional. La autenticación con la plataforma es interna y se gestiona mediante HMAC.

Posibles resultados

OK

Todos los checks han pasado. El script deposita la FLAG en la plataforma. Ve a tu panel y canjéala antes de 30 minutos.

FAIL

Uno o más checks han fallado. El script indica qué medidas no se han aplicado correctamente. No se genera FLAG hasta que todo esté correcto.

ERR

Error al depositar la FLAG en la plataforma. Comprueba tu conectividad a Internet y vuelve a ejecutar el script.

Paso siguiente — canjea la FLAG

Cuando el script termina con éxito, la FLAG ya está depositada. Entra en tu panel, localiza la sección "Canjear FLAG" y pégala para obtener los puntos.

// La FLAG tendrá este aspecto

STB-linux-hardening-1716048000-a3f8b2c14d5e6789

// Caduca a los 30 minutos — un solo uso

Nota: si la máquina ya fue completada por tu cuenta, el canje será rechazado y no se registrarán puntos duplicados.

# 7. Sistema de Puntos y Ranking

Cada máquina tiene asignada una puntuación según su nivel de dificultad. Los puntos se acumulan en tu perfil al canjear la FLAG correspondiente y determinan tu posición en el ranking global.

Dificultad Puntos base
Fácil 50 pts
Media 100 pts
Difícil 200 pts
→

Los puntos se asignan al canjear la FLAG desde el panel, no al ejecutar el script. Asegúrate de canjearla antes de que caduque.

→

Ranking global: los 5 mejores agentes aparecen en la página principal. Tu posición se actualiza inmediatamente tras el canjeo.

→

Sin repeticiones: cada máquina solo puede completarse una vez por cuenta. Si intentas canjear una FLAG de una máquina ya completada, será rechazada.

→

Historial: en tu panel puedes ver el listado completo de máquinas completadas con los puntos obtenidos en cada una.

# 8. Referencia de la API

La plataforma expone una API REST utilizada internamente por el script de evaluación. Puedes consultarla si quieres integraciones propias o depurar el comportamiento del script.

POST /api/v1/flags/registrar/

Deposita una FLAG firmada con HMAC-SHA256. Llamado automáticamente por el script de evaluación cuando el hardening es correcto.

// Body (JSON)

{

"flag": "STB-linux-hardening-1716048000-a3f8b2c14d5e6789",

"machine_name": "linux-hardening"

}

// Respuesta exitosa (201)

{

"message": "FLAG registrada correctamente"

}

POST /api/v1/flags/canjear/

Canjea una FLAG previamente depositada. Requiere sesión web activa. Otorga los puntos al usuario autenticado.

// Body (JSON)

{

"flag": "STB-linux-hardening-1716048000-a3f8b2c14d5e6789"

}

// Respuesta exitosa (200)

{

"message": "FLAG canjeada correctamente",

"points": 100

}

POST /api/v1/get_user_data

Devuelve el perfil del usuario asociado a la API Key: nombre, puntos totales y máquinas completadas.

// Headers

X-API-KEY: <tu_api_key>

// Respuesta exitosa (200)

{

"username": "agent47",

"puntos_totales": 350,

"maquinas_completadas": ["web-basics", "linux-hardening"]

}

Para una documentación interactiva completa visita /api/docs/ (requiere permisos de administrador o entorno de desarrollo).

# 9. Preguntas Frecuentes

El script termina con OK pero no aparece ninguna FLAG en el panel.

Recarga la página del panel. Si la FLAG sigue sin aparecer, comprueba que el script tiene conectividad con la plataforma y que no hay un error silencioso en la respuesta del endpoint /api/v1/flags/registrar/.

He tardado más de 30 minutos en canjear la FLAG y ya no es válida.

Las FLAGS caducan a los 30 minutos por seguridad. Vuelve a ejecutar el script de evaluación para generar una nueva, siempre que el hardening siga aplicado correctamente.

¿Puedo completar la misma máquina varias veces para ganar más puntos?

No. Si intentas canjear una FLAG de una máquina que ya completaste, el sistema la rechazará. Cada máquina solo suma puntos una vez por cuenta.

No veo los puntos en mi panel después de canjear la FLAG.

Los puntos se actualizan al instante. Si no aparecen, recarga la página. Si el canjeo devolvió un error, revisa el mensaje: puede ser FLAG caducada, ya usada o máquina ya completada.

¿Necesito introducir mi API Key en el script?

No. El script usa autenticación HMAC interna y no requiere ningún parámetro de usuario. Simplemente ejecútalo con python3 check_hardening.py.

El contenedor Docker no arranca. ¿Qué hago?

Ejecuta docker-compose logs para ver los errores. Asegúrate de tener los puertos requeridos libres y de que el daemon de Docker está activo (docker info).
