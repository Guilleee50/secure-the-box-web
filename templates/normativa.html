Secure the Box | Rules of Engagement (RoE)

Última actualización: 29/04/2026

1. Introducción y Propósito

Bienvenido al entorno de entrenamiento y validación de Secure the Box. Este documento define las Reglas de Enfrentamiento (RoE) y las políticas de uso aceptable para todos los Agentes SOC (usuarios) registrados en la plataforma.

La participación en este entorno de laboratorio implica la aceptación incondicional de estas normas. El objetivo de esta plataforma es estrictamente educativo y de evaluación de competencias en fortificación de sistemas (Hardening) y respuesta a incidentes.

2. Alcance Permitido (In-Scope)

Las únicas acciones autorizadas dentro de la plataforma son:

  • Acceso y auditoría de las máquinas virtuales etiquetadas explícitamente como "Entorno de Laboratorio" o "Máquina Vulnerable".
  • Ejecución de scripts de validación proporcionados por la plataforma (`check_hardening.py`) dentro de los entornos desplegados.
  • Modificación de configuraciones dentro de las máquinas objetivo con el único fin de mejorar su postura de seguridad (Hardening).

3. Restricciones Estrictas (Out-of-Scope)

Queda TERMINANTEMENTE PROHIBIDO:

  • Ataques a la Infraestructura Core: Intentar comprometer, escanear o realizar ataques de denegación de servicio (DDoS) contra el servidor web principal, la base de datos MariaDB o la infraestructura subyacente de AWS.
  • Manipulación de Puntuaciones: Falsificar peticiones HTTP/API, alterar el envío de la api_key o intentar sumar puntos por máquinas no resueltas legítimamente.
  • Interferencia con otros Agentes: Atacar, monitorizar o interferir en los entornos de laboratorio de otros usuarios registrados.
  • Uso de Malware Real: Desplegar ransomware, troyanos o herramientas destructivas no controladas dentro del entorno de laboratorio.

4. Monitorización y Privacidad

Como Agente SOC, entiendes y aceptas que todas tus acciones en la plataforma están monitorizadas. Registramos direcciones IP, uso de API Keys y telemetría de los scripts de validación con el fin de garantizar la integridad del sistema y evaluar el rendimiento académico. No almacenes información personal, contraseñas reales ni datos sensibles en las máquinas del laboratorio.

5. Consecuencias por Infracción

Cualquier violación de estas Reglas de Enfrentamiento resultará en la revocación inmediata de tu API Key, la expulsión de la plataforma y la notificación a los administradores del sistema (Tribunal Evaluador) para la aplicación de las sanciones académicas o legales correspondientes.
